Description

In this programming experiment, we're building a generator that takes an image and reimagines it using shapes such as squares, triangles, and circles. Furthermore, we automatically cycle through multiple permutations of this generated image, which are then evaluated and scored. Below are the 4 images that were used to generated the "Evolutionary Impressions".

First Image: Bridge over a Pond of Water Lilies by Claude Monet Second Image: Endwalker Credits Art by SquareEnix Third Image: Japanese Breakfast picture taken by Ethan Lin Fourth Image: "This is Fine" Meme by KC Green

Technical

  • Using the drop-down menu, we are able to switch between our inspiring images.
  • Shapes are generated depending on the images' "Shape Tag". The placement of the shapes are randomized and its fill color is picked based on the location color of the original image. The shape sizes are also randomized depending on the "Detail Amount"
  • The "Change Opacity" and "Detail Amount" buttons make use of preset ranges and values to change the fill opacity or change the number of generated shapes respectively.
  • The "Re-Color" button inverts the colors of each shape by subtracting the RGB values from 255.
  • The mutation slider determines the rate at which we make incremental changes to the design. More specifically, the lower the slider is, the more drastic the changes are to the shapes' size and locations.

Reflection

This experiment was all about understanding what each value and variable did. Many of the innovations and integrations relied heavily on changing certain values and variables. Additionally, this experiment taught me a lot about making code efficient since efficient code would lead to more "fps". Keeping a reasonable amount of fps was the most difficult part of this experiment. There were many times when I wanted to implement or try something new, but since complicated tasks would lower the fps, I had to think of different ways and ideas.
